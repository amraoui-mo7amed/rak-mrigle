# المواصفات التقنية: منصة راك مريقل (Rak-Mrigle)

## 1. تعريف المشروع

- **اسم المشروع:** راك مريقل (Rak-Mrigle)
- **العميل:** أسامة
- **المطور الرئيسي:** عمراوي محمد
- **مدة المشروع:** 6 أسابيع (كحد أقصى)

## 2. المكونات التقنية والبيئة البرمجية
- **الخلفية (Backend):** Python / Django (إطار عمل حديث لضمان القابلية للتوسع)
- **الواجهة الأمامية (Frontend):** HTML5, CSS3, JavaScript (Vanilla), Bootstrap 5, HTMX
- **الأيقونات:** FontAwesome 6 (النسخة المجانية)
- **الترجمة والتدويل:** Django i18n (دعم كامل من اليمين إلى اليسار RTL للغة العربية)
- **إستراتيجية الهاتف المحمول:** ويب متجاوب (الهاتف أولاً / إمكانية PWA)

## 3. المتطلبات الوظيفية الأساسية
### 3.1 أدوار المستخدمين والمصادقة
- **المشرف (Admin):** الإشراف الكامل على المنصة، التحقق من المستخدمين، ومراقبة المعاملات.
- **الزبون (Customer):** تدفق يعتمد على الطلبات للنقل أو الآلات الثقيلة.
- **المزود (Provider):** إدارة الملف الشخصي والأسطول، التعامل مع الطلبات، وتسوية العمولات.
- **منطق المصادقة:** تسجيل الدخول عبر **الشبكات الاجتماعية** أو **البريد الإلكتروني** مع **الموافقة اليدوية على الحساب**.

### 3.2 دليل الخدمات
- **النقل:** شاحنات صغيرة/كبيرة، مركبات نفعية (مثل رينو ماستر).
- **الآلات:** جرافات، رافعات، معدات رفع، آلات البناء.
- **الخدمات اللوجستية:** سحب السيارات (Dépannage)، الخدمات اللوجستية حسب الطلب.

### 3.3 نظام التسعير والعمولات
- **منطق المسافة:** السعر = المسافة (كم) × السعر لكل كيلومتر.
- **الأجرة الأساسية:** حد أدنى للتكلفة يعتمد على نوع المركبة.
- **منطق الساعة:** سعر الساعة + تكاليف الوقود/المشغل الاختيارية.
- **تكاليف إضافية:** وقت الانتظار / صعوبات الوصول.
- **هيكل العمولات:** 1% (ترويجي - الشهر الأول)، 10% (نمو - من الشهر الثاني وما بعده).
- **التسوية:** دفع العمولات أسبوعياً (كل يوم أحد) عبر CCP أو نقداً.

### 3.4 ربط الخدمة والتواصل
- **سير عمل المزود:**
    - ملف شخصي مخصص للمستخدم.
    - إنشاء/حذف/تحديث عرض.
    - قبول/رفض طلبات الزبائن.
    
- **سير عمل الزبون:** 
    - تصفح العروض.
    - تقديم طلب مع تقدير السعر.
    - قبول المزود مع عرض سعري.
    - قبول/رفض الزبون.
    - إتمام الطلب.
    
- **سير عمل المشرف:**
    - إدارة المستخدمين.
    - سجل العروض/الطلبات.
    - إحصائيات عامة.
    
- **الخصوصية:** رقم الهاتف مقفل حتى التأكيد المتبادل.

## 4. البنية التقنية (جاهزة للتطوير)
### 4.1 كيانات قاعدة البيانات (نماذج Django)
- **المستخدمون (Users):** AbstractUser ممتد للتحكم في الوصول القائم على الأدوار (RBAC).
- **المزودون (Providers):** مرتبطون بالمستخدمين؛ يتضمن حالة التحقق من رخصة السياقة.
- **الفئات (Categories):** هيكل هرمي للنقل والآلات.
- **المركبات (Vehicles):** مرتبطة بالمزودين؛ تخزن الفئة والسعة والصور.
- **الطلبات (Orders):** تتبع مكان الاستلام/الوجهة، الحالة (قيد الانتظار/نشط/تم)، والتسعير.
- **المدفوعات (Payments):** سجل لديون العمولات الأسبوعية ومدفوعات المزودين.
- **التقييمات (Ratings):** نظام 1-5 نجوم مع ملاحظات اختيارية لبناء الثقة.
- **التنبيهات (Notifications):** تنبيهات فورية لتتبع إجراءات المستخدمين.

### 4.2 التوطين (i18n)
- **اللغات:** 
    - العربية 
    - الفرنسية
    - الإنجليزية (🇬🇧)

## 5. إرشادات واجهة المستخدم (UI/UX)
- **الألوان الأساسية:** 
    - الأصفر الذهبي (#FFD700) للطاقة والحيوية.
    - الأزرق الداكن (#1A2B3C) للثقة والاحترافية.
    - الرمادي الفاتح (#F5F5F5) للحدود والخلفيات.
- **الأسلوب:** 
    - بسيط (Minimalist).
    - أداء عالي السرعة.
    - سهل الاستخدام لغير التقنيين.

## 6. الجوانب القانونية والأمنية
- **المسؤولية:** بند "وسيط فقط" صارم في شروط الخدمة (ToS).
- **التحقق:** مراجعة وثائق المزود من طرف المشرف (الهوية، الرخصة، التأمين).

## 7. الجدول الزمني للتطوير (6 دورات سريعة - أسبوع لكل دورة)
- **الأسبوع 1:** إعداد البيئة، النماذج الأساسية (المستخدم، الفئات)، والمصادقة.
- **الأسبوع 2:** ملفات المزودين، إدارة المركبات، ومنطق توثيق اعرف عميلك (KYC).
- **الأسبوع 3:** محرك التسعير (مسافة/ساعة) وسير عمل المطابقة (طلب/تقدير).
- **الأسبوع 4:** نظام التنبيهات الفورية، منطق خصوصية جهات الاتصال، وسير عمل الطلب.
- **الأسبوع 5:** تنفيذ تعدد اللغات (i18n)، تحسين واجهة RTL، ولوحة تحكم المشرف.
- **الأسبوع 6:** سجل العمولات، اختبار قبول المستخدم النهائي (اختبار الجزائر العاصمة)، ونشر المشروع.
